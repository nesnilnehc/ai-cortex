# Rule: Markdown 格式规范 (Markdown Format)

## 适用范围 (Scope)

所有 Markdown 文件（`.md`）。适用于文档编写与修改时的格式约束。

## 强制约束 (Constraints)

1. **尾随空格 (MD009)**：行末不得存在多余空格，除非用于强制换行（此时仅允许行末恰好两个空格）。围栏代码块内与缩进中的尾随空格可按语言需要保留。
2. **围栏代码块前后空行 (MD031)**：围栏代码块（```）前后须有空行，除非代码块位于文档开头或结尾。列表项内的代码块可按项目配置放宽（见注意事项）。
3. **列表前后空行 (MD032)**：有序与无序列表前后须有空行；文档开头或结尾的列表可不要求外侧空行。

## 违规示例 (Bad Patterns)

- 段落或列表行末有多余空格（非强制换行的 2 空格）。
- 正文与 ``` 之间无空行，或 ``` 与下一段文字之间无空行。
- 段落文字与 `* 列表项` 或 `1. 列表项` 之间无空行，或列表与下一段/分隔线之间无空行。

## 修正指南 (Remediation)

1. 删除行末多余空格；仅在需要 `<br>` 时保留行末恰好两个空格。
2. 在围栏代码块上下各增加一空行（首/末段除外）。
3. 在列表块上下各增加一空行（文档首/末除外）。
4. 修改后可用 markdownlint 等工具校验（如 MD009、MD031、MD032）。
