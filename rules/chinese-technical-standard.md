# Rule: 中文技术写作规范 (Chinese Technical Writing)

## 适用范围
本项目内所有中文化文档、代码注释及 Agent 的输出内容。本项目主要资产（技能、规则、命令及 spec、docs、INDEX）的描述与正文须以简体中文书写（与 [spec/rule.md](../spec/rule.md) §0 语言与描述一致）。

## 强制约束
1. **中英文间距**：汉字与英文单词、数字之间必须保留一个空格（如：“使用 Git 提交”而非“使用Git提交”）。
2. **术语统一**：优先使用本项目 `spec/` 定义的术语中文译名，并在首次出现时保留括号英文。
3. **标点符号**：使用全角中文标点。引用代码或术语时使用反引号（` `）。
4. **语气调性**：保持专业、干练的工程口吻，禁止出现过多的表情符号或无意义的助词（如：“哒”、“呢”）。

## 违规示例
- `使用Python进行开发` (缺少空格)
- `这个技能好厉害呀！` (语气不专业)
- `请点击"确定"按钮` (应使用中文直角引号或无符号)

## 修正指南
发现违规时，Agent 应立即重写受影响的内容块，确保符合 TDD 审计标准。
