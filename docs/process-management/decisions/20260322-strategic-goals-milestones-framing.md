# 战略目标与里程碑的层次与语义

**状态**：已批准  
**日期**：2026-03-22  
**范围**：strategic-goals.md、roadmap.md（含里程碑节点）的定义与关系。

---

## 1. 问题

当里程碑 M1–M4 均标记为 Done 时，逻辑上可被理解为「项目已完成」。但 AI Cortex 是持续演进的技能目录与生态，不存在「项目结束」。需澄清：

- 战略目标与里程碑的语义边界
- 「里程碑 Done」与「项目持续」如何共存
- 目标、里程碑、roadmap 的层次关系

---

## 2. 设计决策

### 2.1 战略目标：持续优化的结果愿景

**定位**：详见 [strategic-goals.md](../../project-overview/strategic-goals.md)

核心理解：战略目标是**持续演进的方向**，而非有截止期的交付物。

**设计意义**：允许项目在长期保持方向一致，同时通过里程碑实现分阶段的检查点。

### 2.2 里程碑：分阶段的检查点

**定位**：详见 [roadmap.md](../roadmap.md)

核心理解：里程碑是**有阶段的、可验证的检查点**，而非最终目标。

**设计意义**：支撑短期的里程碑交付与长期的目标演进并行。

### 2.3 目标与里程碑的关系

| 层次 | 含义 | 示例 |
| :--- | :--- | :--- |
| **战略目标** | 持续追求的结果方向 | 交付链能力可获得、采纳摩擦低 |
| **里程碑** | 某阶段在该方向上的可验证检查点 | M1 可发现性基线（Phase 1） |
| **Roadmap** | 有时间维度的计划与举措 | Evolution Roadmap、CHANGELOG 版本 |

- 一个目标可对应多个阶段的多个里程碑
- 一个里程碑可推进一个或多个目标
- 里程碑 Done 后，目标仍在；后续里程碑继续推进同一目标
