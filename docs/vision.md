# 愿景与哲学 (Vision & Philosophy)

**定位**：AI Native 时代的 **Spec 驱动的 Skills/Rules/Commands 资产库**——可复用能力 + 规范与测试保障质量。详见 [架构说明](architecture.md)。

---

## 为什么需要 AI Cortex？

Prompt 已是“软代码”，但现状是：**碎片化**（逻辑散落于 `.txt` 或字符串）、**难复用**（夹杂项目黑话）、**难审计**（能力边界不可验证）。

## 核心理念

用 `spec/` 规约 Skills/Rules/Commands，使其**可维护、可协作、可测试**。

- **Spec 驱动**：编写与运行时契约 → 可发现、可注入、可验证。
- **可测试**：每 Skill 配标准用例与自检，与 Spec 一致。
- **可发现**：`INDEX.md` 做能力目录与元数据。

### 目标用户

| 角色 | 说明 |
| :--- | :--- |
| 开发者/最终用户 | 通过 Agent 读入口与索引，用自然语言或 /command 使用。 |
| 作者/贡献者 | 按 spec 编写与测试，提交 PR。 |
| 集成方 / CI | 解析 manifest 与入口，集成到 IDE、Agent 或流水线。 |

### 差异化

- **不是**单纯提示词仓库：有 Spec、自检与测试。
- **不是**项目私有黑话包：资产受 Spec 约束，可版本化、可追溯。
- **提供** `INDEX.md` 能力目录，便于检索与调度。
