## 何时扩展

在以下情形添加结构化需求文档（如 `docs/requirements-planning/core-v2.md`）：

- v2.2+ 或重大范围变更
- `analyze-requirements` 技能产出经验证且值得持久化的需求
- 多干系人需要正式需求追溯

在撰写前使用 `analyze-requirements` 将模糊意图转化为经验证、可测试的需求。
