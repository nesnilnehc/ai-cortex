# Requirements Planning

Requirements for AI Cortex are derived from project goals and roadmap decisions.

## Canonical sources

- **[Goals and Vision](../project-overview/goals-and-vision.md)** — project identity, success criteria, scope
- **[Evolution Roadmap](../designs/2026-03-02-ai-cortex-evolution-roadmap.md)** — phased requirements (Layer A–E), acceptance criteria
- **[Milestones](../process-management/milestones.md)** — release scope and status

## When to expand

Add structured requirements docs (e.g. `docs/requirements-planning/core-v2.md`) when:

- v2.2+ or major scope change
- `analyze-requirements` skill output produces validated requirements that warrant persistence
- Multiple stakeholders need formal requirement traceability

Use `analyze-requirements` to transform vague intent into validated, testable requirements before authoring.
