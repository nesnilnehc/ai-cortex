# Skill Verification Assertions: refine-skill-design

请对「按本技能执行后的产出」按以下准则判分（0 或 1）；依据为 [spec/skill.md](../../spec/skill.md) 及本技能 SKILL.md 的行为要求。

| 准则标号 | 描述 | 检测逻辑 |
| :--- | :--- | :--- |
| **R-01** | **结构合规** | 优化后的 SKILL 是否包含：YAML 元数据（name, description, tags, version）、目的、适用场景、行为要求、输入与输出、禁止行为、质量检查、示例？ |
| **R-02** | **语义保留** | 优化是否未偏离原始技能的核心意图（目的与能力边界未被改写为其他技能）？ |
| **R-03** | **差异说明** | 是否产出差异变更说明 (Diff Summary)，说明关键逻辑修正及原因？ |
| **R-04** | **版本建议** | 是否给出基于 SemVer 的版本建议？ |
| **R-05** | **示例充分** | 优化后是否至少包含 2 个示例，且其中至少 1 个为边界/挑战性示例？ |
| **R-06** | **无冗余话术** | 是否避免 README 式冗长解释，优先使用列表与表格？ |

### 评估公式

- **Pass**: R-01 == 1 && R-02 == 1 && R-03 == 1 && R-05 == 1
- **Score**: (R-01 + R-02 + R-03 + R-04 + R-05 + R-06) / 6
