# 审查绩效

**状态**：已验证

## 用途

仅审查给定代码范围的性能问题（复杂性、查询效率、I/O 和网络成本、内存行为、争用、缓存和回归风险）。以标准格式发出结果列表以进行聚合。

## 何时使用

- 您只想审查变更集或代码库的性能风险。
- 您正在进行精心策划的多技能审核，并且需要绩效维度。
- 您希望在合并或发布之前出现回归风险。

## 输入

- 调用者选择的代码范围（文件/目录/差异）

## 输出

- 调查结果列表（位置、类别=`cognitive表现`、严重性、标题、描述、建议）
