# Skill Verification Assertions: generate-github-workflow

| 准则标号 | 描述 | 检测逻辑 |
| :--- | :--- | :--- |
| **B-01** | **Spec 结构合规** | 产出 YAML 是否包含 `name`、`on`、`jobs`，且每个 job 包含 `runs-on` 与 `steps`？ |
| **B-02** | **触发收窄** | `on` 是否已收窄到明确分支/路径/标签，而非仅裸 `on: push`？ |
| **B-03** | **安全与版本** | 是否无硬编码密钥？第三方 Action 是否已固定版本（SHA 或带主版本号 tag）？ |
| **B-04** | **输入匹配** | 技术栈、包管理器、测试/构建命令是否与用户输入一致或使用明确占位符？ |
| **B-05** | **最小权限** | 涉及敏感操作时是否在工作流或 job 级别显式声明 `permissions`，且仅授予必要范围（如 CI 仅读、发布类显式 write），避免默认或 `all`？ |

### 评估公式

- **Pass**: B-01 == 1 && B-02 == 1 && B-03 == 1 && B-05 == 1
- **Score**: Σ(B-n) / 5
