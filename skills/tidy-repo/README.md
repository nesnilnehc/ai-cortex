# 整仓整理 (tidy-repo)

一次性审计仓库目录结构——检测错放文件、命名不一致、空目录和过期制品；输出优先级整理报告；可选地应用安全、可逆的清理操作。

## 用途

1. **结构审计**：扫描整个仓库（或指定范围），枚举文件和目录；识别不符合项目约定的结构问题。
2. **问题分类**：按类别（错放、命名、空目录、过期制品、重复）和严重性归类所有发现。
3. **清理计划**：按影响和操作代价排序，给出具体、可操作的整理步骤。
4. **安全应用（可选）**：在 `apply-safe` 模式下，仅执行可逆操作（移动文件、删除空目录），危险操作一律跳过并提示人工确认。

## 何时使用

- 仓库积累了大量文件，目录结构感觉混乱
- 新成员加入，需要对齐目录规范
- 项目经历重构，残留大量旧路径或废弃文件
- CI/CD 中周期性结构健康检查
- 与 `assess-docs` 联用：文档规范检查 + 整体仓库结构整理

## 输入

- 仓库根路径（默认：当前工作目录）
- 可选范围过滤（子目录或 glob，如 `src/`、`**/*.bak`）
- 可选模式：`report-only`（默认）| `apply-safe`

## 输出

- 整理报告写入 `docs/calibration/repo-tidy.md`（除非请求快照，否则覆盖）
- 单份报告包含：问题发现列表（位置、类别、严重性、描述、建议）、清理优先级计划、可选应用操作日志

## 安装

```bash
npx skills add nesnilnehc/ai-cortex --skill tidy-repo
```

## 相关技能

- `assess-docs` — 文档规范合规性与准备情况评估
- `review-codebase` — 代码架构与技术债务审查
- `align-planning` — 任务后漂移检测与重新校准
- `commit-work` — 整理后提交变更
